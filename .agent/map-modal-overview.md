# MapModal 介面功能總覽 (Agent 指南)

此文件旨在告知後續 AI Agent 關於 `MapModal` 組件的完整功能，避免操作過於單一。

## 核心分頁
1. **行程規劃 (Itinerary Tab)**:
   - **功能**: 直接搜尋特定地點，或查看起終點之間的 AI 順路推薦。
   - **適用場景**: 已知要去哪裡，或需要 AI 填補行程空隙時。

2. **探索與推薦 (Explore Tab)**:
   - **功能**: 以地圖中心為基準，搜尋周邊景點/餐廳。
   - **子功能**:
     - **Google 模式**: 傳統關鍵字 + 類別分組搜尋。
     - **AI 模式**: 透過敘述（Prompt）讓 Gemini 過濾推薦地點。
   - **適用場景**: 抵達某處後不知道附近有什麼，或有特殊需求（如：慶生餐廳）時。

## 常見交互細節
- **地點同步**: 在探索分頁搜尋地點時，右側地圖會自動移動到對應位置。
- **Hover 連動**: 在列表移動游標時，地圖上的 Marker 會變色放大。
- **自動壓縮**: 搜尋出結果後，搜尋面板會自動收縮以騰出空間顯示列表。

## 開發者提示
- 此 Modal 透過 `useTripStore` 的 `addOptionToNode` 動作與主行程同步。
- 所有的地點資料結構需包含 `id`, `name`, `rating`, `lat`, `lng` 等欄位。
