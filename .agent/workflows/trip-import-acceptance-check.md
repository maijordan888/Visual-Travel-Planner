# 行程匯入 / 建立後驗收 Workflow

本 workflow 用於完成一份行程後的最後檢查，尤其適用於多日行程、跨城市行程、含備選方案與手動交通時間的任務。

## 基本資料

1. 確認行程標題與使用者要求一致。
2. 確認開始日期與結束日期正確。
3. 確認 Sidebar / day switcher 顯示正確天數。
4. 逐日切換 Day，確認日期標籤沒有錯位。

## 起終點

1. 檢查每一天起點。
2. 檢查每一天終點。
3. 住宿日需確認起點與終點是否符合飯店安排。
4. 最後一天終點需符合機場、車站或使用者指定終點。

## 景點節點

1. 逐日檢查 confirmed 景點名稱。
2. 確認每個景點都在正確 Day 與正確順序。
3. 確認每個 confirmed 節點有 Google 地點資料。
4. 優先檢查是否有圖片；沒有圖片時需判斷是 Google 無照片，還是建立方式錯誤。
5. 對餐廳分店、飯店、車站、機場需特別確認名稱是否為正確分店或地點。

## 備選方案

1. 檢查未定項目是否使用同一節點的備選列表呈現。
2. 確認互斥選項沒有散成多個 confirmed 節點。
3. 檢查目前採用的主選項是否合理。
4. 在最終回覆列出仍待決定的備選項目。

## 交通與路線

1. 確認每段交通模式符合使用者要求；未指定時預設 `transit`。
2. 點擊重點路段的 `查看路線`，確認 Google Maps 可開啟。
3. 對 API 估算不可靠路段依 `.agent/workflows/manual-route-time-check.md` 手動回填分鐘數。
4. 長途段落需檢查交通工具名稱，例如新幹線、特快車、Skyliner、機場線。
5. 確認手動時間重新整理後仍保留。

## 保存與回報

1. 若使用者要求雲端保存，使用畫面上的同步 / 行程庫操作保存。
2. 確認 UI 顯示已保存或已同步。
3. 重新整理頁面，確認資料仍存在。
4. 最終回覆包含：
   - 行程名稱與日期。
   - 建立方式是否全程 UI-only。
   - 已確認的圖片 / Google 地點資料狀態。
   - 已手動確認的交通段落。
   - 任何 UI 限制造成的差異或未完成項。
