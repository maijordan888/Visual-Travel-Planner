# 手動確認交通時間 Workflow

本 workflow 用於日本地鐵、大眾運輸、特快車、新幹線、機場交通等 API 估算可能不可靠的路段。

## 原則

- 每段交通工具預設使用 `transit`。
- 一般市區移動以 Google Maps `查看路線` 建議的大眾運輸時間為準。
- 長途段落需在節點註記或最終回覆中標明交通類型，例如「新幹線」或「Skyliner」。
- 不直接修改 store 或 API 回傳值；只使用 UI 的手動交通時間輸入。

## 操作流程

1. 切換到要檢查的 Day。
2. 找到要確認交通時間的目標節點。時間顯示代表「從上一站到此站」。
3. 確認交通模式圖示選在大眾運輸 / `transit`。
4. 點擊該段旁邊的 `查看路線`。
5. 在新開啟的 Google Maps 頁面確認起點、終點與交通模式。
6. 以 Google Maps 建議的最適合路線時間為準，必要時參考使用者指定的出發時間。
7. 回到 Travel Planner App。
8. 點擊交通時間旁的手動編輯按鈕或 `修正時間`。
9. 輸入分鐘數，例如 1 小時 35 分輸入 `95`。
10. 確認 UI 顯示 `手動: X 分鐘` 或等效手動時間標記。

## 長途段落特別檢查

- 飯店 / 出發地 → 主要車站。
- 新幹線或特快車路段，例如名古屋站 → 東京站。
- 轉乘後到住宿地的最後一段。
- 機場交通，例如日暮里站 → 成田機場 Skyliner。
- 回程當天需保留航班、報到與尖峰時間緩衝。

## 驗證

- `查看路線` 能成功開啟 Google Maps。
- Google Maps 顯示的起點與終點符合該段行程。
- 回填後 UI 使用手動時間，而不是 fallback 估算時間。
- 重新整理頁面後，手動分鐘數仍保留。
- 最終回覆列出已人工確認的長途段落與分鐘數。
