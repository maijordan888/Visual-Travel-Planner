# Google PlacePicker 新增景點 Workflow

本 workflow 用於新增已知景點、餐廳、車站、飯店或機場。目標是讓節點取得 Google Place 資料，而不是只建立文字節點。

## 操作流程

1. 切換到要加入景點的 Day。
2. 在時間軸上找到插入位置。
3. 點擊畫面上的 `新增第一個景點` 或節點之間的 `在後面新增景點` / `+` 按鈕。
4. 若出現空白待決定節點，點擊該節點上的 `開啟地圖/交由 AI 推薦`。
5. 在地圖 modal 中確認位於「行程規劃」搜尋介面。
6. 點擊 `地圖搜尋景點或餐廳...` 輸入框。
7. 輸入地點名稱。跨語系地點可優先使用 Google Maps 常見英文或日文名稱，以提高命中率。
8. 等待 Google 建議結果出現。
9. 點選正確的 Google 建議結果，不要只按 Enter 留下自由文字。
10. 確認地圖或結果卡顯示正確地點後，點擊 `確認加入`。
11. 回到行程時間軸，確認新節點出現在正確位置。

## 圖片與資料驗證

每個用此流程加入的 confirmed 節點都要檢查：

- 節點顯示地點名稱。
- 節點有縮圖；若 Google 沒有照片，需在回覆中明確說明是 Google 地點無照片，而非未讀入 Google 資料。
- 節點有地址、評分、類型標籤或其他 Google 地點資訊之一。
- 點擊 `查看路線` 可以開啟 Google Maps。
- 重新整理頁面後，節點仍保留圖片與地點資訊。

## 常見問題

- 如果節點沒有圖片，先確認是不是使用了純文字匯入或 API 寫入。UI-only 任務遇到這種情況應重做該節點。
- 如果搜尋建議結果不準，改用更完整名稱，例如加上城市、分店名、車站名或英文名。
- 如果加入後位置不正確，不要直接改資料；用 UI 刪除該節點後重新新增。
- 如果 `確認加入` 後沒有反應，先等待地圖 modal 完成載入，再檢查是否已選中 Google 建議結果。
